# 🤖 Kazan Events Digest Bot

Telegram-бот для автоматической отправки дайджеста мероприятий Казани каждый понедельник в 10:00.

## 📋 Возможности

- 📅 Автоматическая отправка дайджеста каждый понедельник в 10:00
- 🏃 Поддержка множества категорий мероприятий (спорт, деловые события и т.д.)
- 🔍 Парсинг мероприятий из Telegram-каналов
- 📍 Информация о дате, времени и месте проведения
- 🔗 Прямые ссылки на мероприятия

## 🚀 Установка

### 1. Клонирование репозитория

```bash
git clone https://github.com/erepinkazan/kazani-events-digest.git
cd kazani-events-digest
```

### 2. Установка зависимостей

```bash
pip install -r requirements.txt
```

### 3. Настройка переменных окружения

Создайте файл `.env` или настройте GitHub Secrets:

```bash
BOT_TOKEN=YOUR_BOT_TOKEN
CHAT_ID=YOUR_CHAT_ID
```

## 🔐 GitHub Secrets

Для работы бота нужно установить следующие secrets в GitHub:

1. `BOT_TOKEN` - токен Telegram бота (от @BotFather)
2. `CHAT_ID` - ваш личный Telegram ID

**Как установить secrets:**

1. Перейдите в Settings репозитория
2. Secrets and variables → Actions
3. New repository secret
4. Добавьте `BOT_TOKEN` и `CHAT_ID`

## 📅 Расписание

Дайджест отправляется:
- **День:** Каждый понедельник
- **Время:** 10:00 (Moscow Time)

Вы можете запустить бот вручную через GitHub Actions:
1. Перейдите в Actions
2. Выберите "Send Weekly Digest"
3. Нажмите "Run workflow"

## 🛠️ Расширение функциональности

### Добавление нового канала

1. Откройте `config.py`
2. Добавьте новый канал в словарь `CHANNELS`:

```python
'new_channel': {
    'name': 'Название канала',
    'category': '🎯 КАТЕГОРИЯ',
    'url': 'https://t.me/new_channel',
    'keywords': ['ключевое слово 1', 'ключевое слово 2', ...]
}
```

### Изменение расписания

Отредактируйте `.github/workflows/send_digest.yml`:

```yaml
on:
  schedule:
    - cron: '0 10 * * 1'  # Понедельник в 10:00
```

[Справка по cron синтаксису](https://crontab.guru/)

## 🐛 Решение проблем

### Бот не отправляет сообщения

1. Проверьте, что токен и chat_id верны
2. Проверьте, что бот запущен и активен
3. Посмотрите логи в GitHub Actions

### Мероприятия не парсятся

1. Проверьте, что каналы доступны
2. Убедитесь, что ключевые слова правильные
3. Посмотрите логи парсера
